# 概念解剖：渐进式披露 (Progressive Disclosure)

> Tags: #UX #Skill-Design #Cognitive-Load
> Date: 2026-02-20

## 预处理

- **澄清定义**：这是一种交互设计策略，旨在通过仅向用户展示当前步骤所需的最关键信息，来降低认知负荷。在 Skill 编写中，意味着不一次性堆砌所有字段（如订单详情），而是根据用户所处的阶段（查询、确认、支付、售后）分层展示。
- **识别关键词**：认知边界、分层、时机 (Timing)、降噪。

## 八维探索

### 历史溯源
起源于 1980 年代的计算机人机交互研究，核心是为了解决专家功能与新手学习曲线之间的矛盾。随着移动端小屏幕和 AI 聊天交互（有限的单次输出容量）的兴起，它从“可选”变成了“必选”。

### 辩证分析
- **正题**：完整性（一次展示所有，用户拥有全局掌控感）。
- **反题**：专注度（只展示当前，用户拥有极高的操作效率）。
- **合题**：动态披露（系统预测或响应用户意图，只展示“恰好”够用的完整性）。

### 现象学还原
想象你在餐馆点菜。优秀的侍者（Skill）会先给你看主菜单，确认肉类后再问你几分熟（订单明细），而不是一上来就把厨师的采购单、调料比例和厨房库存全部报给你。

### 语言学解构
- **Progressive (渐进)**：强调时间序列上的流动性。
- **Disclosure (披露)**：揭开面纱。在订单语境下，披露是一种“真相”的逐步同步。

### 数学形式化
- $CognitiveLoad = \sum I_i$ (其中 $I$ 为信息项)。
- **目标**：$Max(I_{useful}) / \sum (I_{total}) \to 1$，且单次交互的 $CognitiveLoad < Threshold$。
- 逻辑：将一个高维张量（完整订单）投影到低维子空间（当前操作界面）。

### 存在主义审视
这是一种对“有限性”的尊重。承认人的注意力和短期记忆是有限的资源，不以信息的冗余来冒犯用户的生命时间。

### 美学维度
其美感在于**“呼吸感”**。信息不再是倾倒而下的沙石，而是随着对话节奏律动的波浪。
- 意象：一张折叠的密信，阅读一行，展开一行。

### 元哲学反思
这种策略隐含了一个假设：系统（AI）比用户更清楚用户现在需要什么。这是一种“家长式”的温柔，但也可能造成信息孤岛。

## 内观与提炼

- **从概念内部看世界**：如果你是“渐进式披露”，你觉得一下子把订单 ID、Sku、物流单号、供应商信息全吐出来的行为是极度无礼的。你倾向于先推开一扇窗，等对方点头，再打开一扇门。
- **提炼核心**：**权力下放的时机化。**

## 顿悟压缩

1. **The One 公式**：`体验 = (核心操作 + 1层关联信息) / 剩余信息的隐藏`
2. **一句话**：在订单管理中，先给结果，再给细节，最后给选项。
3. **拓扑图**：
```bash
    [ 用户意图: 查订单 ]
             |
    +--------V--------+
    | 12. 概览层      |  <-- 状态: 已成交 / 金额: 100
    +-----------------+
             | (确认/需要详情?)
    +--------V--------+
    | 13. 明细层      |  <-- Sku / 数量 / 规格
    +-----------------+
             | (物流? 退款?)
    +--------V--------+
    | 14. 操作层      |  <-- 物流跟踪 / 申请售后
    +-----------------+
```
